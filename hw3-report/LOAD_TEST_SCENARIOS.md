# Нагрузочное тестирование: сценарии

## Цель
Проверить производительность и устойчивость ML API под различной нагрузкой, а также собрать метрики:
- RPS по эндпоинтам
- latency p50 / p95 / p99
- error rate (4xx + 5xx)
- время инференса модели p50 / p95

## Инструмент
- Locust (`tests/load/locustfile.py`)
- Тестируемые эндпоинты:
  - `GET /health`
  - `GET /models`
  - `GET /datasets`
  - `POST /predict`

## Профиль нагрузки

### Сценарий 1: Smoke
- Пользователи: 5
- Spawn rate: 1 user/sec
- Длительность: 2 минуты
- Цель: проверить корректность и базовую доступность API

### Сценарий 2: Baseline
- Пользователи: 30
- Spawn rate: 5 users/sec
- Длительность: 10 минут
- Цель: измерить штатный RPS и latency в нормальном режиме

### Сценарий 3: Stress
- Пользователи: от 30 до 120
- Spawn rate: 10 users/sec
- Длительность: 15 минут
- Цель: найти точку деградации по p95/p99 и росту error rate

### Сценарий 4: Spike
- Пользователи: резкий рост 10 -> 100
- Spawn rate: 25 users/sec
- Длительность: 5 минут
- Цель: оценить реакцию системы на всплеск трафика
